# 随机图片 API - 使用指南

## 📁 项目已创建

项目路径：[d:\coding\zidong\random-image-api](file:///d:/coding/zidong/random-image-api)

## 🚀 使用步骤

### 第一步：创建 GitHub 仓库

1. 打开 GitHub 并登录：https://github.com
2. 点击右上角 "+" → "New repository"
3. 填写信息：
   - Repository name: `random-image-api`
   - Description: `随机图片 API`
   - 选择 Public（公开）
   - 不需要勾选任何初始化选项
4. 点击 "Create repository"

### 第二步：上传文件

#### 方法 1：直接上传（推荐新手）

在创建好的空仓库页面：
1. 点击 "uploading an existing file"
2. 将 `random-image-api` 文件夹中的**所有文件**拖拽到上传区域
3. 确保文件结构正确：
   ```
   .github/workflows/update-list.yml
   images/.gitkeep
   js/images.js
   index.html
   .gitignore
   ```
4. 点击 "Commit changes"

#### 方法 2：使用 Git 命令行

```bash
cd random-image-api
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/random-image-api.git
git push -u origin main
```

### 第三步：上传图片

1. 进入仓库的 `images/` 文件夹
2. 点击 "Add file" → "Upload files"
3. 将你的 .jpg 或 .jpeg 图片拖拽上传
4. 点击 "Commit changes"

### 第四步：启用 GitHub Pages

1. 进入仓库 → Settings
2. 左侧菜单找到 "Pages"
3. Source 部分：
   - 选择 "Deploy from a branch"
   - Branch 选择 "main"，文件夹选择 "/ (root)"
4. 点击 "Save"
5. 等待 1-2 分钟

### 第五步：访问你的网站

访问：`https://你的GitHub用户名.github.io/random-image-api/`

## 🔧 自动更新机制

当你上传新图片到 `images/` 文件夹时：

1. GitHub Actions 会自动运行
2. 扫描所有 .jpg/.jpeg 文件
3. 自动更新 `js/images.js` 中的 `IMAGE_PATHS` 数组
4. 更新网站内容

## ❓ 常见问题

### Q: 网站显示 404 错误？

等待 2-3 分钟后再试，GitHub Pages 需要时间部署。

### Q: 图片没有显示？

1. 检查图片是否在 `images/` 文件夹中
2. 检查图片格式是否为 .jpg 或 .jpeg
3. 确认 GitHub Actions 是否成功运行

### Q: 如何删除图片？

直接在 GitHub 仓库中删除图片文件，然后推送即可。

### Q: 如何查看 GitHub Actions 状态？

进入仓库 → Actions 标签页，可以看到自动构建的日志。
